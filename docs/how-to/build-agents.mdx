## Build Your Custom Agent

### Understand the Template Agent

The template includes a simple greeting agent in `src/beeai_agents/agent.py`:

```python
@server.agent(
    metadata=Metadata(ui={"type": "hands-off"})
)
async def example_agent(...):
    ...
```

It uses environment variables, handles messages, and responds with a polite greeting.

### Wrap Your Own Agent

Letâ€™s say you want to integrate a CrewAI agent.

<Accordion title="Example: Wrap a CrewAI Agent">

```python
@server.agent(
    metadata=Metadata(
        ui={"type": "hands-off"},
        framework="CrewAI",
        env=[{
            "name": "OPENAI_API_KEY",
            "description": "OpenAI API key for the CrewAI agent",
            "required": True
        }]
    )
)
async def crewai_agent(input: list[Message], context: Context) -> AsyncGenerator[RunYield, RunYieldResume]:
    ...
```

</Accordion>

**Tips for Wrapping Agents:**
- The function name becomes the agent ID (override with `name=`)
- The docstring becomes the description
- Use `input[-1]` to get the latest user message
- Send output with `yield MessagePart(content="...")`
- Multiple agents? Just add more decorated functions.

<Tip>
Each `@server.agent` function defines a new agent service.
</Tip>

### Update Dependencies

Add your agent's dependencies to `pyproject.toml`:

```toml
dependencies = [
    "acp-sdk>=0.8.1",
    "requests>=2.31.0",     # Add your dependencies here
    "crewai >= 0.30.0"      # For example, CrewAI
]
```

Then sync with `uv sync`.

## Deploy Your Agent

### Option 1: Build Locally

```bash
beeai build .
beeai list  # Confirms it's registered
beeai run your_agent "test input"
```

### Option 2: Deploy from GitHub

1. Push to GitHub

```bash
git add .
git commit -m "Add custom agent"
git push origin main
```

2. Register via BeeAI:

```bash
beeai add https://github.com/your-username/your-agent-repo
```

### Option 3: CI/CD with GitHub Actions

The starter includes a built-in GitHub Actions workflow:
1. Push your changes
2. GitHub builds a Docker image
3. Image is published to GHCR (GitHub Container Registry)

Then add your agent:

```bash
beeai add ghcr.io/your-username/your-agent:latest
```
